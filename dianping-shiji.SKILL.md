# 大众点评食记写作 Skill

## 触发条件

用户输入店名（如"帮我写XX店的食记"、"XX餐厅"、"XX面馆怎么写"），或明确提到"大众点评食记""探店笔记""美食评价"时自动触发。

## 核心原则（四不四要）

### 四不
1. **不幻想**：所有菜品、价格、环境细节必须来自真实评论或公开信息，绝不凭空编造
2. **不夸大**：不用"天花板""封神""绝绝子""此生必吃"等过度吹捧词
3. **不文绉绉**：禁止书面语堆砌、成语连排、排比句轰炸
4. **不AI腔**：禁用"总体而言""值得一提的是""不得不说""众所周知"等模板句式

### 四要
1. **要真实**：像普通食客饭后随手写的感受，有优点也有小遗憾
2. **要白话**：口语化、短句为主（单句≤25字为主流）、段落≤3行
3. **要SEO**：店名完整出现3-4次，区域+路名+地标各至少1次，菜品名自然穿插
4. **要有用**：读者看完能决定去不去、怎么去、点什么

## 写作流程

### 第一步：搜集真实信息

用 web_search 搜索以下关键词（每个至少搜一条）：
- `{店名} 大众点评` → 获取评分、人均、推荐菜
- `{店名} 怎么样` → 获取真实食客评价
- `{店名} 地址 怎么走` → 获取到达方式和周边地标

### 第二步：提取关键素材

从搜索结果中提取：
- 3-5个真实菜品名及价格区间
- 2-3条真实食客评价中的原话或感受
- 到达方式（地铁几号线哪站、公交哪路、附近什么显眼建筑）
- 周边地标（商场、学校、写字楼、景点）

### 第三步：结构化写作

按以下结构输出（500-700字）：

---

**段落1：怎么去（60-80字）**
地铁X号线XX站X口出来，往XX方向走大概X分钟。路边有个XX（显眼地标），看到它就快到了。店在XX路XX号，门口长什么样。

**段落2：第一印象（60-80字）**
什么时间去的，店里人多不多，环境大概什么样。不用细描，一两句带过。

**段落3-5：菜品实感（200-300字，核心）**
挑2-3个菜，每个菜写一段：
- 菜名（不用引号括，直接说）
- 分量、卖相（一句话）
- 味道真实感受（好就说好，一般就说一般，有槽点也可以提）
- 价格（自然带出，不要列价格表）

示例写法：
> 糖醋里脊上来的时候还在滋滋冒油。面衣薄，肉感足，不是那种一口下去全是面粉的。酸甜口偏甜，北方人应该会喜欢。38一份，两个人够吃。

**段落4：周边彩蛋（50-80字）**
吃完可以在附近逛逛XXX，或者走两步就是XXX商场。如果是专门跑一趟，建议顺便把XXX也去了。

**段落5：一句话总结（30-50字）**
什么人适合来、什么场景来最合适。不要强行推荐，实事求是指出适合人群。

## 语言风格细则

### 句式要求
- 短句占70%以上（≤15字）
- 禁止连续3句以上相同句式
- 偶尔用"就是""反正""说实话""怎么说呢"这类口语词破节奏

### 禁用词清单
**AI味禁用词**：总体而言、值得一提的是、不得不说、众所周知、在…中、不仅…而且、既…又、通过…实现
**浮夸禁用词**：天花板、封神、绝绝子、此生必吃、惊艳、灵魂、治愈、宝藏
**书面禁用词**：饕客、珍馐、唇齿留香、回味无穷、食指大动

### 标点规则
- 句号为主，少用感叹号（全文≤2个）
- 不用分号
- 并列用顿号或逗号，不用破折号
- 引用食客原话用「」不用""

### 必含SEO要素
- 店名完整出现3-4次
- 区域名（如朝阳区/徐汇区）+ 路名
- 最近地铁站名
- 1个周边地标
- 2-3个菜品关键词

## 输出前自检

完成初稿后，逐条检查：
1. [ ] 有没有编造没搜到的菜品或价格？
2. [ ] 有没有出现禁用词？
3. [ ] 单句有没有超过30字的？（有就拆）
4. [ ] 读起来像不像真人饭后随手写的？
5. [ ] 字数在500-700之间？
6. [ ] SEO要素都埋进去了吗？
7. [ ] 到达方式和周边标的物写了吗？
